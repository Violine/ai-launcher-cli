# Экран загрузки и прогресс (по ТЗ)

## Что в ТЗ и спецификации

### SPECIFICATION.md

- **5.2.2. Экран загрузки** — показывается при получении списка моделей из API:
  - Заголовок: `LOADING`
  - Текст: `Loading available models...`
  - **Прогресс-бар:** `████████████░░░░░░░░░░░░░░░░░░░░░░░`
  - Строка статуса: `/ Please wait...`

- **FR-202** (раздел 3.2): отображение индикатора загрузки с анимацией (при запросе к API за моделями).

### Где переиспользовать

Один и тот же экран/компонент с прогресс-баром можно использовать для:

| Сценарий | Текст / статус | Прогресс |
|----------|----------------|----------|
| Загрузка списка моделей (FR-202) | Loading available models... | индикатор без процентов (анимация) или спиннер |
| Скачивание обновления launcher (3.6) | Downloading update... | 0–100% по байтам |
| Обновление MCP (3.7) | Updating MCP package X... | по пакетам или общий прогресс |
| Обновление конфигов агентов | Updating configs... | по файлам или спиннер |

Итог: в TUI нужен **общий экран/виджет прогресса**: заголовок, описание, прогресс-бар (0–100% или неопределённый) и строка статуса.

## План реализации

1. **Общий компонент в TUI**  
   Экран (или виджет) с:
   - заголовком (например, «LOADING» или «DOWNLOAD»);
   - строкой описания («Loading...», «Downloading update...»);
   - прогресс-баром: либо 0–100% (для скачивания), либо неопределённый (анимация/спиннер для «ждём ответ API»);
   - строкой статуса внизу (меняемый текст).

2. **updater.Install**  
   Опциональный callback прогресса `func(written, total int64)` при скачивании (по `Content-Length` и записанным байтам), чтобы:
   - в CLI выводить текстовый прогресс в stderr;
   - в TUI (когда установка запускается из меню) обновлять экран прогресса.

3. **CLI**  
   При `ai-launcher autoupdate` и установке передавать в Install callback, который пишет в stderr строки вида `\r  [=====>    ] 45%`.

4. **TUI при установке из меню**  
   После подтверждения «Install» переключаться на экран прогресса (спиннер или 0% → 100%), запускать Install в goroutine; по завершении получать одно сообщение и переходить к экрану «Готово» или ошибки. Позже можно подключить callback из п.2 для отображения реального процента.

5. **FR-202 и MCP**  
   Тот же экран: при запросе моделей — неопределённый прогресс/спиннер; при обновлении MCP — прогресс по пакетам или один общий прогресс-бар.

## Файлы для правок

- `internal/tui/` — новый экран (например, `ScreenProgress`), виджет прогресс-бара (Lip Gloss), обновление модели по сообщениям прогресса.
- `internal/updater/updater.go` — сигнатура Install с опциональным `onProgress func(written, total int64)` и передача прогресса при `io.Copy`.
- `internal/modules/autoupdate/` — при запуске из TUI переходить на экран прогресса и вызывать Install с callback; при запуске из CLI — передавать callback для вывода в stderr.
- Дальше: экран загрузки моделей (FR-201/202) и сценарии MCP — использовать тот же экран/виджет.
